## Project Overview

Android port of the Orthodox Calendar iOS app. Native Kotlin + Jetpack Compose. Multi-locale Orthodox Church calendar with daily saints, scripture readings, fasting guidance, and saint biographies. Fully offline ‚Äî all data bundled as JSON in assets.

## Build

```bash
# Build debug APK
./gradlew assembleDebug

# Install on connected device/emulator
./gradlew installDebug

# APK location
app/build/outputs/apk/debug/app-debug.apk
```

Requires `local.properties` with `sdk.dir=/path/to/Android/sdk`.

## Data Files

All in `app/src/main/assets/localization/`:
- `sr.json`, `ru.json`, `en.json` ‚Äî UI strings
- `calendar_{sr,ru,en,en_nc}_{2024-2030}.json` ‚Äî 28 calendar data files

Same JSON format as the iOS app. Generated by the Python pipeline in the iOS repo (`scripts/shared/build_database.py`).

## Key Design Patterns

### Dark Mode
`AppColors` uses `LocalIsDarkTheme.current` (not `isSystemInDarkTheme()`) so it respects the app's theme setting (SYSTEM/LIGHT/DARK). The `OrthodoxCalendarTheme` provides `LocalIsDarkTheme` via `CompositionLocalProvider`.

### Localization
No Android string resources ‚Äî all UI text comes from bundled JSON files loaded by `LocalizationManager`. Language switching reloads the bundle and triggers recomposition via `CalendarUiState.localization`.

### ViewModel State
Single `CalendarUiState` data class exposed via `StateFlow`. Contains: currentMonth, currentYear, daysInMonth, viewMode (LIST/GRID), language, theme, localization bundle, scrollToTodayTrigger.

### CalendarDay Model
- `dayOfWeek`: Python convention (0=Mon, 6=Sun), NOT Android convention
- `weekdayIndex`: converts to localization array index (0=Sun)
- `Feast.moveable`: distinguishes algorithmic feasts from fixed saints
- `Feast.description`: expandable text for major feasts

### Saint Bio Matching (DayDetailScreen)
Same logic as iOS ‚Äî `commonWords` filter, single-bio Serbian mode, keyword-match multi-bio EN/RU with used-bio tracking. Moveable feasts never get bios. `Feast.description` takes priority over bio.

### Haptic Feedback
`Haptics` utility uses `View.performHapticFeedback()`:
- `medium` ‚Äî Today button
- `light` ‚Äî Search, Settings, Share, date picker
- `selection` ‚Äî Month arrows, grid/list toggle, grid day tap

### Navigation
Compose Navigation with `NavGraph`. Routes: Splash ‚Üí Calendar ‚Üí DayDetail, Calendar ‚Üí Search/Settings/About. DatePicker shown as `ModalBottomSheet` from CalendarTabScreen.

### Share & Reminders
- Share: `Intent.ACTION_SEND` with plain text
- Reminders: `Intent.ACTION_INSERT` to `CalendarContract.Events.CONTENT_URI`

## Locales

Four locales: `SR`, `RU`, `EN`, `EN_NC` (New Calendar).

`EN_NC` shares `en.json` but loads `calendar_en_nc_*.json` data. Fixed feasts on Gregorian dates (Dec 25 = Christmas). All `when` expressions use `AppLanguage.EN, AppLanguage.EN_NC ->` for English text.

## Common Pitfalls

1. **AppColors composable getters**: All adaptive colors are `@Composable get()` properties. Cannot be called from non-composable functions ‚Äî pass colors as parameters or make the function `@Composable`.
2. **JSON in assets, not resources**: Calendar files go in `assets/localization/`, not `res/raw/`. Loaded via `context.assets.open()`.
3. **CalendarDay identity**: Use `gregorianDate` as key in `LazyColumn`. When locale changes, recompose by changing the list key.
4. **MonthHeaderBar callbacks**: `onPreviousMonth`/`onNextMonth` are `() -> Unit`. The IconButton `onClick` wraps them with haptics ‚Äî don't double-wrap.
5. **Fasting colors**: `fastStrictBg`, `fastWaterBg`, etc. are `@Composable` getters. Use them inside composable scope only.
